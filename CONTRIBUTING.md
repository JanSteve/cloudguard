# Contributing to CloudGuard

Thank you for your interest in contributing to CloudGuard! 🛡️

## How to Contribute

### Reporting Issues
- Found an inaccuracy in the security content? Open an issue.
- Spotted a typo or broken link? Open an issue.
- Have a suggestion for a new section? Open an issue.

### Pull Requests
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/add-new-service`)
3. Make your changes to `index.html`
4. Test locally by opening `index.html` in your browser
5. Submit a pull request with a clear description

### Content Guidelines
- Write explanations in plain English — no unexplained jargon
- Include real-world analogies where possible
- Cite official AWS documentation for technical claims
- Keep exam relevance ratings accurate

### What We're Looking For
- New AWS security services to document
- Additional security checklists
- Corrections to existing content
- Translations into other languages
- Accessibility improvements

## Code of Conduct
Be respectful, be constructive, be helpful. We're all here to learn.
